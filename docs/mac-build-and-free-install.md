## TianlaiVotePad 免费安装小白操作手册

### 1. 先说结论

如果你没有付费 Apple Developer 账号，只使用普通 Apple ID，也就是 Xcode 里的 `Personal Team`，你可以做到：

- 在 Mac 上打开这个项目
- 编译项目
- 在 iPad 模拟器里运行
- 安装到你自己手上的 iPad 上测试

但你做不到：

- 导出正式分发用的 `.ipa`
- 通过 `Ad Hoc` 分发给 10 台固定 iPad
- 用 `TestFlight` 给别人装
- 在 GitHub Actions 里免费打出真机可分发安装包

所以，这份文档的目标不是“正式分发”，而是“让完全不懂代码的人，也能把项目先在自己的 iPad 上跑起来”。

### 2. 这份文档适合谁

这份文档适合：

- 第一次接触 Xcode
- 没有写过 iOS 代码
- 只想先把项目在 iPad 上安装出来
- 只接受“免费方法”

这份文档不适合：

- 要给 10 台固定 iPad 做长期稳定安装
- 要导出安装包给别人发
- 要上架 App Store

### 3. 你要准备的东西

在开始之前，请先确认你有下面这些东西。

#### 3.1 必须有

- 一台 Mac
- 一部 iPad
- 一根可以传数据的数据线
- 一个普通 Apple ID
- 已下载好的本项目代码

#### 3.2 建议有

- 稳定网络
- 较新的 macOS
- 较新的 Xcode
- iPadOS 16 或以上

#### 3.3 如果你还没有安装 Xcode

去 Mac 的 `App Store` 搜索 `Xcode` 并安装。

说明：

- Xcode 体积很大，下载会比较久
- 第一次打开 Xcode 时，系统可能会要求安装额外组件，直接点确认即可

### 4. 你最终会做成什么样

如果一切正常，最后你会看到：

- 项目在 Xcode 里可以打开
- 项目能在 iPad 模拟器里运行
- 项目能安装到你自己的 iPad
- iPad 上能点开这个 App

### 5. 第一步：把项目打开

#### 5.1 找到项目文件夹

先找到这个仓库所在的文件夹，也就是：

`JavaSpringFirst`

里面会有一个 Xcode 工程文件：

`TianlaiVotePad.xcodeproj`

#### 5.2 打开工程

你有两种打开方式。

方法 A：双击打开

1. 在 Finder 里进入项目目录
2. 双击 `TianlaiVotePad.xcodeproj`

方法 B：终端打开

```bash
cd JavaSpringFirst
open TianlaiVotePad.xcodeproj
```

#### 5.3 第一次打开时要做什么

第一次打开后，Xcode 可能会做这些事情：

- 索引项目
- 读取文件
- 检查依赖
- 弹窗要求安装组件

你只需要：

- 等待
- 有弹窗就点确认

当左上角不再一直转圈时，说明基本准备好了。

### 6. 第二步：先在模拟器里跑起来

建议你不要一上来就装真机，先在模拟器里跑一次，能更快判断项目本身有没有问题。

#### 6.1 选择运行目标

在 Xcode 顶部中间区域，你会看到：

- 左边是运行按钮，一个三角形
- 中间是当前 Scheme，一般显示 `TianlaiVotePad`
- 旁边是设备选择框

点击设备选择框，选一个 iPad 模拟器，例如：

- `iPad (10th generation)`
- `iPad Air`

#### 6.2 点击运行

点击左上角三角形运行按钮，或者按：

```text
Command + R
```

#### 6.3 成功时你会看到什么

如果成功：

- 会自动打开一个 iPad 模拟器窗口
- 模拟器里会安装并启动 App
- 你会看到投票器首页

#### 6.4 如果模拟器没起来

先别慌，常见情况有：

- Xcode 还在初始化，继续等一会儿
- 选错了设备，重新选 iPad 模拟器
- Xcode 太旧，更新后再试

只有模拟器能跑起来之后，再继续装真机。

### 7. 第三步：把 Apple ID 登录到 Xcode

免费安装到真机，核心就是让 Xcode 用你的 Apple ID 给 App 做临时签名。

#### 7.1 打开账号设置

在屏幕顶部菜单栏点击：

`Xcode > Settings > Accounts`

#### 7.2 添加 Apple ID

1. 点击左下角 `+`
2. 选择登录 Apple ID
3. 输入你的 Apple 账号和密码
4. 如果需要验证码，照提示输入

#### 7.3 登录成功后怎么看

登录成功后，你应该能在左边看到你的 Apple ID。

点开后，通常会看到一个团队，名字里一般会包含：

`Personal Team`

这就说明免费账号方式可以继续了。

### 8. 第四步：给项目设置签名

这是最关键的一步。

#### 8.1 进入项目设置

在 Xcode 左侧文件栏最顶部，点击蓝色项目图标：

`TianlaiVotePad`

然后在中间区域：

1. 选中左侧 `PROJECT` 下的 `TianlaiVotePad`
2. 再选中右侧 `TARGETS` 下的 `TianlaiVotePad`

#### 8.2 打开签名页面

点击上方标签：

`Signing & Capabilities`

#### 8.3 需要设置的内容

你至少要检查这三项：

1. `Automatically manage signing`
   这里要打勾

2. `Team`
   这里选择你刚刚登录的 `Personal Team`

3. `Bundle Identifier`
   这里必须是一个唯一值

#### 8.4 Bundle Identifier 怎么写

你可以写成类似这样：

```text
com.yourname.tianlaivotepad
```

例如：

```text
com.zhangsan.tianlaivotepad
com.lisi.votepad
com.companyname.tianlai.vote
```

原则只有一个：

- 不能和别人重复

#### 8.5 如果提示 Bundle Identifier 已被占用

那就把它改成更独特一点，例如：

```text
com.你的拼音.当前日期.tianlaivotepad
```

例如：

```text
com.zhangsan.20260329.tianlaivotepad
```

### 9. 第五步：连接你的 iPad

#### 9.1 用线连接

用数据线把 iPad 连到 Mac。

注意：

- 有些线只能充电不能传数据
- 如果连接后 Xcode 完全看不到设备，优先怀疑线的问题

#### 9.2 在 iPad 上信任这台电脑

第一次连接时，iPad 可能弹出：

`要信任此电脑吗？`

你需要点击：

- `信任`

然后输入 iPad 锁屏密码。

#### 9.3 在 Mac 上确认设备是否被识别

识别成功后，你通常能在 Xcode 顶部设备列表里看到你的 iPad 名称。

如果没看到：

1. 拔线重插
2. 解锁 iPad
3. 再等几十秒

### 10. 第六步：打开 iPad 的开发者模式

有些新系统上，真机运行前需要打开开发者模式。

在 iPad 上依次进入：

`设置 > 隐私与安全性 > 开发者模式`

如果看到了这个开关：

1. 打开它
2. 按提示重启设备
3. 重启后再次确认开启

如果你没看到这个入口，也不一定有问题，可以先继续后面的步骤。

### 11. 第七步：把 App 安装到 iPad

#### 11.1 在 Xcode 里选择你的 iPad

回到 Xcode 顶部设备列表，把模拟器改成你的真实 iPad。

你会看到设备名称，类似：

- `张三的 iPad`
- `iPad`

#### 11.2 点击运行

点击左上角三角形运行按钮，或者按：

```text
Command + R
```

#### 11.3 第一次安装时会发生什么

第一次装机会比较慢，Xcode 通常会依次完成：

- 编译
- 签名
- 传输到设备
- 安装
- 启动 App

你只需要等待，不要频繁拔线。

#### 11.4 安装成功时会看到什么

如果成功：

- iPad 主屏幕会出现 App 图标
- App 会自动打开，或者你可以手动点开
- Xcode 顶部状态区域不再报红错

### 12. 第八步：信任开发者证书

第一次安装后，如果 iPad 上打开 App 时提示：

`未受信任的开发者`

请在 iPad 上这样操作：

1. 打开 `设置`
2. 进入 `通用`
3. 进入 `VPN 与设备管理`
4. 找到对应的开发者账号
5. 点击 `信任`

然后再回桌面重新打开 App。

### 13. 最常见报错和处理方法

这一节非常重要，小白一般最容易卡在这里。

#### 13.1 报错：No Signing Certificate / Signing failed

意思：

- 没有正确签名

解决方法：

1. 确认你已经在 `Xcode > Settings > Accounts` 登录 Apple ID
2. 确认 `Signing & Capabilities` 里的 `Team` 选了 `Personal Team`
3. 确认 `Automatically manage signing` 已勾选
4. 重新运行一次

#### 13.2 报错：Bundle Identifier is not available

意思：

- 这个包名已经被别人用了

解决方法：

把 `Bundle Identifier` 改成一个更独特的新值。

#### 13.3 报错：Developer Mode required

意思：

- iPad 没开开发者模式

解决方法：

去 iPad 打开：

`设置 > 隐私与安全性 > 开发者模式`

#### 13.4 报错：Device is locked

意思：

- iPad 当前锁屏

解决方法：

- 解锁 iPad
- 保持亮屏
- 再试一次

#### 13.5 报错：Could not locate device support files

意思：

- Xcode 版本太旧，不支持当前 iPad 系统

解决方法：

- 更新 Xcode 到更新版本

#### 13.6 App 装上了，但点不开

常见原因：

- 还没信任开发者证书
- 免费账号签名已过期

解决方法：

1. 先去 `VPN 与设备管理` 信任
2. 如果还是不行，回到 Xcode 再重新运行安装一次

### 14. 如果你想再装到第二台、第三台 iPad

免费账号下，不是“分发”，而是“逐台连接、逐台安装”。

操作方法就是重复上面的真机安装流程：

1. 拔掉当前 iPad
2. 换另一台 iPad
3. 信任电脑
4. 如有需要，开启开发者模式
5. 在 Xcode 顶部选中这台 iPad
6. 再点一次运行

但是你要注意：

- 这不是正式商用分发方式
- 维护起来很麻烦
- 免费签名通常 7 天左右会过期
- 不适合 10 台设备长期使用

### 15. GitHub 在免费方案里能帮你做什么

当前仓库已经在 GitHub 上，可以做的事情是：

- 保存代码
- 自动跑模拟器编译

但 GitHub 在免费方案下不能帮你：

- 免费导出正式 `.ipa`
- 免费做真机批量分发

原因很简单：

- 免费 Apple 账号没有正式分发能力
- 你当前仓库里的真机打包工作流依赖证书和描述文件

### 16. 为什么你不能免费稳定装 10 台固定 iPad

这是很多人最容易误解的地方。

免费账号不是“完全不能装真机”，而是：

- 只能做个人测试
- 不适合正式交付
- 没有正规分发能力

如果你真的要让 10 台固定 iPad 长期稳定使用，最终还是要走：

- 付费 Apple Developer Program
- `Ad Hoc` 或 `TestFlight`

### 17. 建议你最实际的操作顺序

如果你现在只想先把项目跑起来，最推荐这样做：

1. 安装 Xcode
2. 打开项目
3. 先跑模拟器
4. 登录 Apple ID
5. 配置 `Personal Team`
6. 修改 `Bundle Identifier`
7. 连接你自己的 iPad
8. 打开开发者模式
9. 点击运行安装
10. 信任开发者证书
11. 打开 App 检查页面是否正常

### 18. 你怎么判断自己已经成功了

只要满足下面 4 条，就说明你已经装成功了：

- Xcode 左上角没有红色报错
- iPad 桌面上出现了 App 图标
- App 可以点开
- 你能看到投票器首页

- 先验证项目在真机上能不能跑

免费方案最不适合做这件事：

- 长期给 10 台固定 iPad 使用

如果你只是现在要验证现场效果，这份文档已经够用。

如果你后面确认要上 10 台固定 iPad，我建议再切回付费方案，不要继续在免费方式上消耗时间。
